代码内容的解读
1. //开头的为注释内容，双斜杠开头的注释，只会注释本行开头到结尾，如果要写多行注释可以使用/*开始 */结束的成对符号，来进行注释，多行注释不能进行嵌套使用。编译器不会对注释部分进行编译与执行，注释的主要作用是为代码提供解释说明，好的注释是代码的重要组成部分，他可以帮助我们在很久以后从新理解我们自己编写的代码，也可以在团队合作中，让他人更容易读懂和接手你的代码
2. Obejective-C是区分大小写的，一个字母在Obejective-C中大写和小写是存在不同的意义的，比如A与a代表的意思是不同的，也就是A不等于a
3. #import部分意思为导入别人已经写好的代码或者包，这里可能是同事所写的代码，也可能是系统的内容。比如我们要处理一个特别大的数字运算，现实中我们可能需要使用计算器，使用计算器帮助我们进行数字运算的时候，就相当于我们在代码中使用#import将别人写好的工具拿过来使用一样，
4. int main() {}部分是在声明一个函数，main函数较为特别，main是在清楚的告诉程序从何处开始执行，小括号中是在指定代码执行完毕后，返回值的类型，大括号中是函数的主体，也就是需要执行的部分，对于返回值将在后面进行学习
5. @autoreleasepool {}是自动释放池，写在这对大括号中的代码，在内存管理部分会有系统进行自动释放，相关内容后续会进行学习
6. NSLog();是一个打印函数，它的作用是显示该函数的执行日期，时间，程序名以及传递给他的参数等内容，当中@表示传递给它的参数是NSString字符串对象，对于程序相关数据类型，后续进行学习
7. Objective-C的所有程序语句必须使用“;”进行结尾，同时需要注意程序中不能出现中文模式下的英文字母和符号，所有代码中的内容都必须是英文输入法下的字母和符号
8. return 0;意思为终止执行main函数，同时返回一个状态值，按照约定，返回0意味着程序是正常结束的，任何非0的返回值都意味着程序可能出现了错误
